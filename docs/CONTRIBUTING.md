# Contributing

## Branch and Commit
- Use conventional commits (`feat:`, `fix:`, `docs:`, `chore:`).
- Keep docs and code updates in one PR when they are coupled.

## Required Checks
- Lint and typecheck must pass.
- Unit tests required for logic changes.
- E2E smoke required for route/page changes.

## AI Docs Sync
If a page behavior changes, update:
- `docs-ai/GEMINI_PAGE_BRIEF.md`
- `docs-ai/GEMINI_MOTION_BRIEF.md`
- `docs-ai/GEMINI_QA_CHECKLIST.md`
